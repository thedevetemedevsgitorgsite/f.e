14. File Structure & Requirements

To ensure buyers can easily implement your work and to maintain DevTemple’s quality standards, all submissions must adhere to the following technical requirements:

  • Organized Directory: Files must be logically organized. Use clear folder naming conventions (e.g., /src, /dist, /assets, /docs). Avoid deeply nested or messy structures that hinder usability.
  • The "How to Use" Document: Every submission must include a README.md or instructions.txt file in the root directory. This document must contain:
    • A brief overview of the project.
    • Installation steps (e.g., npm install).
    • Configuration requirements (e.g., environment variables needed).
    • Basic usage examples or commands.
  • Language & Framework Tagging: The short public description must end with the primary language or framework using hash tags.
    Format: Project Name — Brief value proposition.
    Example: A drag-and-drop productivity dashboard. #React #dashboard #js
  • Clean & Commented Code: While we don't audit every line, code should be legible. Provide comments for complex logic. Submissions that are completely obfuscated (unless for security products) or intentionally unreadable may be rejected.
  • Dependency Management: Include necessary manifest files (e.g., package.json, requirements.txt, composer.json). Do not include the actual dependency folders (like node_modules) to keep file sizes optimized.
  • Asset Licensing: If your project includes third-party assets (images, fonts, icons), you must include a CREDITS.md file listing the licenses for those assets to ensure they are cleared for redistribution.
