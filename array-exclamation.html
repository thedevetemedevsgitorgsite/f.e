The Role of ! in FSCSS Arrays

The exclamation mark is FSCSS's method-trigger operator — the boundary between referencing raw array data and invoking transformations on it.

Core Concept
Without !
@arr.name

References the raw data stored in the array. No transformation is applied — the values are rendered as-is into your CSS output.

With !
@arr.name!

Calls a function to modify the array's data before it is rendered. This is the entry point to the entire array method API.

Common Patterns
01
Triggering Transformations
Required before method names like .unit, .prefix, or .reverse. Without !, the method name is treated as plain text.
✓ Correct
@arr.mySizes!.unit(px)
✗ Incorrect
@arr.mySizes.unit(px)
02
Calculations
Triggers mathematical reductions on an array's contents. The ! activates the reduction pipeline.
fscss 
/* Adds all numbers in the array */
total-cost: @arr.prices!.sum;
03
Merging (Array Concatenation)
When used with a plus sign, ! triggers the merge method, combining an existing array with new values.
fscss 
/* Combines the existing list with a new value */
@arr.list1! + [new_item]
Language Comparison

The ! character carries different semantic weight across CSS-adjacent languages. FSCSS repurposes it as a method-trigger — distinct from its meaning everywhere else.

Context Meaning of ! Example
FSCSS Method Trigger — executes an array function @arr.sizes!.unit(rem)
Standard CSS Importance — overrides specificity color: red !important
JavaScript / Sass Logical NOT — negates a boolean value !isActive
Code Example

Converting an array of base numbers into responsive percentage widths using the .unit method:

sidebar-widths.fscss 
/* Define an array of numeric base values */
@arr base_widths[10, 20, 30]

.sidebar {
  /* '!' triggers the '.unit' method to append '%' to each number */
  width: @arr.base_widths!.unit(%);
}

/* Output:
   .sidebar { width: 10% 20% 30%; }
*/
